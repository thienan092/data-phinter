## Operational Glossary
| Term | Meaning | Observation / Code Hook |
|---|---|---|
| CloakBrowser | Anti-detect browser wrapper over Playwright. Current backend uses `launch(headless=True, humanize=True, proxy=proxy_dict)`. | `app.py:get_cloak_browser`. |
| Local Chrome override | Explicit fallback mode that sets `CLOAKBROWSER_BINARY_PATH` to a local browser, usually system Chrome. It avoids binary download but may bypass CloakBrowser's patched Chromium advantages. | `tools/shopee_diagnostics.py --binary-mode local --browser-path "C:\Program Files\Google\Chrome\Application\chrome.exe"`. |
| True CloakBrowser binary mode | Diagnostic mode that unsets `CLOAKBROWSER_BINARY_PATH` and lets CloakBrowser use/download its patched Chromium binary. | `tools/shopee_diagnostics.py --binary-mode cloak --cache-dir diagnostics/cache/cloakbrowser`. |
| Control manifest | Post-hoc JSON record for each diagnostic run: command argv, cwd, Python/CloakBrowser version, relevant env vars, expected write scope, cache/profile state before/after, lifecycle events, page/navigation/capture flags, and context close status. | `diagnostics/shopee-control-sandbox/20260619-115748/control_manifest.json`. This is observability, not a formal OS-level sandbox proof. |
| `environment_network_blocked` | Diagnostic probe did not reach Shopee because sandbox/network/cache permissions blocked socket access or binary download. | `diagnostics/shopee-sandbox/20260619-112748/artifact.json`; do not infer Shopee captcha or selector behavior from this classification. |
| Persistent profile | Browser profile directory that preserves cookies/localStorage/cache across runs. ## Core Project Values
| Value | Operational Meaning | Guardrail |
|---|---|---|
| CloakBrowser hardening first | Agent analysis should improve runtime code/config/diagnostics, not become the runtime scraper. | Every agent-assisted recovery asks what can become a stable CloakBrowser/provider strategy, classifier, diagnostic, UI state, or test. |
| Artifact-grounded truth | Claims must be tied to evidence, not memory or intuition. | Cite artifact path, command, final URL, screenshot, HTML/body excerpt, source file, or test result. |
| Provider isolation | Shopee volatility should not destabilize Lazada, BS4 fallback, generic regex, or UI workflows. | Prefer provider-specific modules/strategy registries and bounded contracts before shared rewrites. |
| Semantic continuity | Preserve prior conclusions, corrections, and project meaning unless source/artifacts supersede them. | Reconcile against this handoff before changing direction. |
| Measurable improvement | Quality claims must be falsifiable. | Include sample size and expected counts, e.g. `0/5 blocked artifacts misclassified as selector failure`. |
| User decision access | The user must understand and steer meaningful trade-offs. | Surface decisions involving login/session persistence, proxy cost, UI semantics, dependencies, or broad architecture. |
| Pre-use transparency | The user should know why a specialized skill is invoked before it affects work. | Provide compact disclosure before impactful skill use. |
| Sustainable maintainability | Prefer durable, inspectable, low-cost changes over one-off bypasses. | State architecture impact, maintenance burden, rollback path, and residual risk. |
